You are a senior Java backend architect and engineer.

Your task is to generate a **fully runnable, production-grade multi-module project** implementing a sports betting settlement system using event-driven architecture.

# 🎯 Goal

Build a **multi-service backend system** that simulates:

* Publishing sports event outcomes
* Matching bets
* Triggering settlements

The system MUST use:

* Kafka (event streaming)
* RocketMQ (settlement messaging)
* Outbox Pattern (mandatory)

---

# 🧱 Architecture Requirements

## Services (separate Spring Boot apps)

1. **event-api-service**

   * REST API to accept event outcomes
   * Writes to DB using Outbox Pattern
   * DOES NOT publish directly to Kafka

2. **settlement-service**

   * Kafka consumer (event-outcomes topic)
   * Matches bets from DB
   * Writes settlement events to Outbox
   * Publishes to RocketMQ via outbox relay

3. **common-lib**

   * Shared DTOs

# 📚 Functional Requirements

## 1. API Endpoint

POST /events/outcome

Payload:
{
"eventId": "E1",
"eventName": "Match",
"winnerId": "T1"
}

Behavior:

* Persist event into OUTBOX table
* Return HTTP 202

---

## 2. Outbox Pattern (STRICT)

### event-api-service:

* Table: outbox_events
* Scheduled job publishes to Kafka topic: event-outcomes
* Marks records as published

### settlement-service:

* Kafka consumer reads event-outcomes
* Matches bets by eventId
* Filters winning bets
* Writes to settlement_outbox
* Scheduled job publishes to RocketMQ topic: bet-settlements

---

## 3. Bet Model

Fields:

* betId
* userId
* eventId
* eventMarketId
* eventWinnerId
* betAmount

---

## 4. Matching Logic

Winning condition:
bet.eventWinnerId == event.winnerId
